Overview

PlatypusTools is a powerful, all-in-one Windows system utility that combines file management, multimedia editing, system cleanup, security tools, digital forensics, and metadata editing into a single, modern WPF application.

Secure Wipe New

Permanently and irrecoverably destroy files using forensic-grade wiping methods. Data wiped with this tool cannot be recovered by any data recovery software or forensic techniques.

Features

  • File or Folder Target ‚Äî select individual files or entire directories
  • Configurable Pass Count ‚Äî more passes = more secure (1 pass is usually sufficient for modern drives)
  • Rename Before Delete ‚Äî randomizes the filename before deletion to prevent name-based recovery
  • Wipe Free Space ‚Äî overwrite all free space on a drive to destroy previously deleted file remnants
  • Confirmation Prompt ‚Äî optional confirmation before irreversible wipe

Robocopy File Transfer

A powerful graphical interface for Windows Robocopy, providing enterprise-grade file copying and synchronization capabilities.

Key Features

  • Mirror Mode - Create an exact copy of source to destination, removing extra files
  • Copy Mode - Copy files without affecting existing files at destination
  • Move Mode - Move files (copy then delete source)
  • Sync Mode - Synchronize directories bidirectionally

How to Use

  1. Source Folder - Select the source directory to copy from
  2. Destination Folder - Select where files should be copied to
  3. Select Mode - Choose Mirror, Copy, Move, or Sync
  4. Configure Options - Set retry counts, threading, and exclusions
  5. Preview - Click Preview to see what would be changed
  6. Execute - Click Start to begin the operation

üì∫ Playback History & Resume (Continue Watching)

How It Works

  1. Recording ‚Äî When you play a track or video (via the webapp or desktop player), a playback record is created or updated. The record stores the media file path, title, artist, album, media type, and a timestamp.

‚òÅÔ∏è Cloudflare Zero Trust New

Cloudflare Zero Trust (formerly Cloudflare Access) places an identity-aware proxy in front of your remote server. Users must authenticate through Cloudflare's identity provider before their requests reach PlatypusTools.

How It Works

  1. You set up a Cloudflare Access Application for your tunnel domain in the Cloudflare Zero Trust dashboard
  2. When a user navigates to your tunnel URL, Cloudflare prompts them to sign in (via email OTP, Google, GitHub, Entra ID, or any configured IdP)
  3. After successful authentication, Cloudflare adds a signed JWT token to the request:
    • In the Cf-Access-Jwt-Assertion HTTP header
    • In the CF_Authorization cookie
  4. PlatypusTools validates this JWT:
    • Fetches Cloudflare's public signing keys from https://<team>.cloudflareaccess.com/cdn-cgi/access/certs
    • Verifies the JWT signature using RS256
    • Validates the issuer matches https://<team>.cloudflareaccess.com
    • Validates the audience (AUD) matches your application's audience tag
    • Checks token expiration
    • Extracts the user's email from the email claim
    • Optionally checks the email against an allowed list

Setting Up Cloudflare Zero Trust

  1. Go to Cloudflare Zero Trust Dashboard
  2. Navigate to Access ‚Üí Applications ‚Üí Add an application
  3. Choose Self-hosted
  4. Set the application domain to your tunnel URL (e.g., platypus.yourdomain.com)
  5. Configure an Access Policy:
    • Allow users by email domain, specific emails, IdP groups, or other criteria
    • Add multiple rules as needed
  6. After creation, note the Application Audience (AUD) tag from the application's overview page
  7. Note your Team Domain (e.g., myteam from myteam.cloudflareaccess.com)

Configuring in PlatypusTools

  1. Open Settings ‚Üí Remote Server ‚Üí Cloudflare Zero Trust
  2. Check "Enable Cloudflare Zero Trust authentication"
  3. Enter your Team Domain (just the subdomain, e.g., myteam)
  4. Enter the Application Audience (AUD) tag from the CF Access application
  5. (Optional) Enter Allowed Emails ‚Äî comma-separated list of email addresses that are permitted access. Leave blank to accept any email that passes CF Access.
  6. Click Save

Authentication Context (Step-Up Auth)

The validator supports Cloudflare's Authentication Context (acrs claim) for step-up authentication. This allows you to require additional authentication factors for sensitive operations. If a request lacks the required auth context, the server returns a 403 with a RequiresStepUp flag, prompting the client to re-authenticate with stronger factors.

Key Caching

Cloudflare's JWKS (JSON Web Key Set) is cached for 1 hour after each fetch to minimize network overhead. The cache is automatically refreshed on the next validation after expiry.

Tip: Cloudflare Zero Trust is ideal when you're already using Cloudflare Tunnels to expose PlatypusTools remotely. It adds robust identity verification without any changes to the webapp code.

üîë Entra ID Direct Authentication New

Entra ID Direct Authentication validates OAuth 2.0 Bearer tokens issued by Microsoft Entra ID. The webapp uses MSAL.js (Microsoft Authentication Library for JavaScript) to sign users in and attach Bearer tokens to every API request.

How It Works

  1. The webapp loads MSAL.js and calls GET /api/auth/config to retrieve the Entra ID configuration (Client ID, Tenant ID, authority URL, scopes)
  2. MSAL.js initializes a PublicClientApplication with these settings
  3. When the user needs to sign in, MSAL.js opens a popup or redirect to Microsoft's login page
  4. After successful sign-in, MSAL.js receives an access token
  5. The webapp attaches this token to every API call: Authorization: Bearer <access-token>
  6. The PlatypusTools server validates the token:
    • Fetches Microsoft's OIDC discovery document from https://login.microsoftonline.com/{tenantId}/v2.0/.well-known/openid-configuration
    • Retrieves the public signing keys from the JWKS endpoint
    • Validates the token signature, issuer, audience, and lifetime
    • Extracts the user's email from email, preferred_username, or upn claims
    • Optionally checks the email against an allowed list

Setting Up Entra ID Direct Auth

  1. Create an App Registration in Azure Portal (same as the original Entra ID setup above)
  2. Note the Application (client) ID, Directory (tenant) ID, and the API Scope ID (Application ID URI)
  3. In PlatypusTools, go to Settings ‚Üí Remote Server ‚Üí Entra ID Authentication
  4. Enter the Client ID, Tenant ID, and API Scope / Application ID URI
  5. Check "Enable Entra ID authentication for webapp"
  6. (Optional) Enter Allowed Emails to restrict access to specific users
  7. Click Save

Multi-Tenant Support

  • Single-tenant: Set the Tenant ID to your Azure AD tenant GUID. Issuer validation is strict.
  • Multi-tenant / Common: Use common or organizations as the Tenant ID. Issuer validation is relaxed to accept tokens from any tenant.

Accepted Audiences

The validator accepts tokens with any of these audiences:

  • The Client ID directly (e.g., 12345678-1234-1234-1234-123456789012)
  • api://CLIENT-ID
  • api://API-SCOPE-ID

üîÄ Using Both Providers Together

You can enable both Cloudflare Zero Trust and Entra ID simultaneously. In this configuration:

  • Cloudflare Zero Trust is checked first on every request

üîí Secure Password Vault New in v4.0

PlatypusTools includes a secure, locally-encrypted password vault for storing credentials, secure notes, and API keys. The vault is accessible from both the desktop application and the Platypus Remote webapp.

Security Model

  • Encryption: AES-256 encryption with a key derived from your master password using PBKDF2 (Password-Based Key Derivation Function 2)
  • Local Storage: The encrypted vault file is stored at %APPDATA%/PlatypusTools/vault.dat ‚Äî your data never leaves your PC
  • Master Password: Required to unlock the vault. Not stored anywhere ‚Äî only the derived key is held in memory while the vault is unlocked
  • Auto-Lock: The vault automatically locks after a configurable period of inactivity

Multi-Factor Authentication (MFA)

The vault supports optional TOTP-based MFA as a second factor for unlocking:

  1. Set up MFA in Settings ‚Üí Vault Security ‚Üí Enable MFA
  2. Scan the QR code or enter the secret key in your authenticator app (Google Authenticator, Microsoft Authenticator, Authy, etc.)
  3. When unlocking the vault, you'll be prompted for your master password and a 6-digit TOTP code

Each vault item can store:

  • Name ‚Äî A descriptive name (e.g., "GitHub Account", "AWS Access Key")

üîß Troubleshooting

Common Issues

FFmpeg/ExifTool not found

  • Ensure FFmpeg and ExifTool are installed in the Tools folder or system PATH
  • Set the correct path via File ‚Üí Set Default Tools Folder
  • Check that ffmpeg.exe, ffprobe.exe, and exiftool.exe are present
  • The app will auto-detect if these tools are in your system PATH

Windows Defender / SmartScreen Blocking

Windows Defender or SmartScreen may flag PlatypusTools because it uses a self-signed certificate. This is a false positive.

  • Option 1: Allow through Defender
    1. Open Windows Security ‚Üí Virus & threat protection
    2. Under "Current threats", find PlatypusTools and click "Allow"
    3. Or add an exclusion: Protection history ‚Üí Add exclusion ‚Üí Folder ‚Üí Select the PlatypusTools install folder
  • Option 2: Install the certificate (Recommended for organizations)
    1. Download PlatypusTools.cer from the GitHub releases or scripts folder
    2. Right-click the .cer file ‚Üí Install Certificate
    3. Select "Local Machine" ‚Üí Next
    4. Select "Place all certificates in the following store" ‚Üí Browse ‚Üí "Trusted Root Certification Authorities" ‚Üí OK ‚Üí Next ‚Üí Finish
  • Option 3: PowerShell installation (Run as Administrator) 
    Import-Certificate -FilePath "PlatypusTools.cer" -CertStoreLocation Cert:\LocalMachine\Root
For IT Administrators: Deploy the certificate via Group Policy or Intune using the Install-PlatypusToolsCert.ps1 script included in the scripts folder.

Access Denied errors

  • Run PlatypusTools as Administrator
  • Some system folders require elevated privileges
  • Check file/folder permissions

Video/Audio playback issues

  • Ensure Windows Media codecs are installed
  • Try converting the file to a more common format (MP4, MP3)
  • Check that the file is not corrupted

System Audit Fix buttons not working

  • Run PlatypusTools as Administrator for auto-fix features
  • Some issues cannot be auto-fixed and require manual intervention
  • Check the "CanAutoFix" status - only items with this enabled can be fixed

Fullscreen controls not appearing

  • Move your mouse to the bottom 150 pixels of the screen
  • Controls appear on hover and auto-hide after 2.5 seconds
  • Use keyboard shortcuts as an alternative (Space, Escape, Arrow keys)

Log Files

Logs are saved to the Logs folder in the application directory.

Access via Help ‚Üí View Log or check %APPDATA%\PlatypusTools\Logs\

Getting Help

If you encounter issues not covered here:

  1. Check the log files for error details
  2. Ensure all dependencies are installed (FFmpeg, ExifTool)
  3. Try running as Administrator
  4. Reset configuration to defaults (File ‚Üí Reset Configuration)
  5. Report issues on GitHub: github.com/dxsteel/PlatypusTools
