üì± Remote Control & Streaming New

Platypus Remote lets you control your PC's music player or stream audio directly to your phone from anywhere. It works as a Progressive Web App (PWA) through your browser ‚Äî no app store download required.

Getting Started

1. Enable the Remote Server

  1. Open PlatypusTools on your PC
  2. Go to Settings (gear icon or Ctrl+,)
  3. Find the Remote Server section
  4. Toggle Enable Remote Server to ON
  5. The server starts on https://localhost:47392

2. Set Up Cloudflare Tunnel (Access from Anywhere)

To access Platypus Remote from outside your home network (e.g., from your phone on mobile data), you need a Cloudflare Tunnel:

  1. Create a free Cloudflare account and add your domain
  2. Install cloudflared ‚Äî PlatypusTools can download it automatically via Settings
  3. Authenticate: run cloudflared tunnel login in a terminal
  4. Create a tunnel: cloudflared tunnel create platypus-remote
  5. Create a config file at C:\ProgramData\cloudflared\config.yml: 
    tunnel: platypus-remote
credentials-file: C:\Users\YOUR_USER\.cloudflared\TUNNEL_ID.json

ingress:
  - hostname: your-subdomain.yourdomain.com
    service: https://localhost:47392
    originRequest:
      noTLSVerify: true
  - service: http_status:404
  6. Add a DNS route: cloudflared tunnel route dns platypus-remote your-subdomain.yourdomain.com
  7. In PlatypusTools Settings, enable Start Tunnel on Launch ‚Äî the tunnel runs as a background process and auto-starts with Windows via a Registry Run key
Tip: Once configured, the tunnel starts automatically when PlatypusTools launches. You can also start/stop it manually from Settings.

3. Open the PWA on Your Phone

  1. On your phone's browser, navigate to your tunnel URL (e.g., https://your-subdomain.yourdomain.com)
  2. The Platypus Remote app loads automatically
  3. Tap Install (or use "Add to Home Screen" on iOS) to pin it as a native-feeling app

Two Playback Modes

Platypus Remote offers two distinct modes, selectable via the Control PC / Stream Here buttons at the top:

üñ•Ô∏è Control PC Mode

Your phone acts as a remote control for the desktop audio player. All audio plays from your PC speakers.

  • Play/Pause, Next, Previous ‚Äî sends commands to your PC in real-time
  • Seek ‚Äî tap the progress bar to jump to any position
  • Volume ‚Äî adjust PC volume from your phone
  • Shuffle & Repeat ‚Äî toggle shuffle and repeat modes
  • Queue ‚Äî view and manage the PC's playback queue; tap a library track to add it
  • Now Playing ‚Äî shows real-time track info, album art, and progress from the PC

  • Next/Previous ‚Äî cycles through your phone's stream queue or library
  • Seek ‚Äî tap the progress bar to jump position in the streaming audio
  • Volume ‚Äî controls your phone's playback volume
  • Mobile Queue ‚Äî a separate queue just for your phone. Tap a library track row to add it to the phone queue (tap the ‚ñ∂Ô∏è button to play immediately). The queue tab shows "X tracks (phone)" with remove buttons
  • Auto-advance ‚Äî when a track finishes, the next track in your phone queue plays automatically
Tip: In Stream mode, the PC queue and phone queue are completely separate. Adding tracks to the queue won't affect what's playing on your PC.

Authentication (Entra ID / Azure AD)

The Remote Server and PWA client are secured with Microsoft Entra ID (formerly Azure AD) authentication. All API endpoints and the SignalR hub require a valid JWT token ‚Äî nothing is accessible without signing in.

How It Works

  • Server: Uses Microsoft.Identity.Web with JWT Bearer authentication. Every API endpoint and the SignalR hub (/hub/platypus) has .RequireAuthorization().
  • Client (PWA): Uses MSAL (Microsoft Authentication Library) for Blazor WebAssembly. Users sign in via Microsoft redirect flow, receive a token, and use it for all API calls and real-time connections.
  • Rate Limiting: Authenticated users are limited to 100 requests per minute (fixed window) to prevent abuse.

Setting Up Entra ID App Registration

  1. Go to Azure Portal ‚Üí App registrations
  2. Click New registration
  3. Name it (e.g., "PlatypusTools Remote"), set supported account types as needed
  4. Set Redirect URI to https://localhost:47392/authentication/login-callback (type: Single-page application)
  5. After creation, go to Expose an API:
    • Set the Application ID URI (e.g., api://YOUR-CLIENT-ID)
    • Add a scope: access_as_user (admin consent optional)
  6. Note the Application (client) ID and Directory (tenant) ID from the Overview page

Configuring PlatypusTools

Run the configuration script from the project root:

.\Configure-PlatypusRemote.ps1 -ClientId "YOUR-CLIENT-ID"

# Optional: specify tenant (defaults to "common" for multi-tenant)
.\Configure-PlatypusRemote.ps1 -ClientId "YOUR-CLIENT-ID" -TenantId "YOUR-TENANT-ID"

# Optional: set a public URL (Cloudflare Tunnel or custom domain)
.\Configure-PlatypusRemote.ps1 -ClientId "YOUR-CLIENT-ID" -PublicUrl "https://platypus.yourdomain.com"

This updates both the server (appsettings.json) and client (wwwroot/appsettings.json) configuration files with your app registration credentials.

Important: The default Client IDs in the config files are placeholders. Authentication will not work until you register your own Entra ID app and run Configure-PlatypusRemote.ps1.

Common Issues

FFmpeg/ExifTool not found

  • Ensure FFmpeg and ExifTool are installed in the Tools folder or system PATH
  • Set the correct path via File ‚Üí Set Default Tools Folder
  • Check that ffmpeg.exe, ffprobe.exe, and exiftool.exe are present
  • The app will auto-detect if these tools are in your system PATH

Windows Defender / SmartScreen Blocking

Windows Defender or SmartScreen may flag PlatypusTools because it uses a self-signed certificate. This is a false positive.

  • Option 1: Allow through Defender
    1. Open Windows Security ‚Üí Virus & threat protection
    2. Under "Current threats", find PlatypusTools and click "Allow"
    3. Or add an exclusion: Protection history ‚Üí Add exclusion ‚Üí Folder ‚Üí Select the PlatypusTools install folder
  • Option 2: Install the certificate (Recommended for organizations)
    1. Download PlatypusTools.cer from the GitHub releases or scripts folder
    2. Right-click the .cer file ‚Üí Install Certificate
    3. Select "Local Machine" ‚Üí Next
    4. Select "Place all certificates in the following store" ‚Üí Browse ‚Üí "Trusted Root Certification Authorities" ‚Üí OK ‚Üí Next ‚Üí Finish
  • Option 3: PowerShell installation (Run as Administrator) 
    Import-Certificate -FilePath "PlatypusTools.cer" -CertStoreLocation Cert:\LocalMachine\Root
For IT Administrators: Deploy the certificate via Group Policy or Intune using the Install-PlatypusToolsCert.ps1 script included in the scripts folder.

Access Denied errors

  • Run PlatypusTools as Administrator
  • Some system folders require elevated privileges
  • Check file/folder permissions

Video/Audio playback issues

  • Ensure Windows Media codecs are installed
  • Try converting the file to a more common format (MP4, MP3)
  • Check that the file is not corrupted

System Audit Fix buttons not working

  • Run PlatypusTools as Administrator for auto-fix features
  • Some issues cannot be auto-fixed and require manual intervention
  • Check the "CanAutoFix" status - only items with this enabled can be fixed

Fullscreen controls not appearing

  • Move your mouse to the bottom 150 pixels of the screen
  • Controls appear on hover and auto-hide after 2.5 seconds
  • Use keyboard shortcuts as an alternative (Space, Escape, Arrow keys)

Log Files

Logs are saved to the Logs folder in the application directory.

Access via Help ‚Üí View Log or check %APPDATA%\PlatypusTools\Logs\

Getting Help

If you encounter issues not covered here:

  1. Check the log files for error details
  2. Ensure all dependencies are installed (FFmpeg, ExifTool)
  3. Try running as Administrator
  4. Reset configuration to defaults (File ‚Üí Reset Configuration)
  5. Report issues on GitHub: github.com/dxsteel/PlatypusTools
